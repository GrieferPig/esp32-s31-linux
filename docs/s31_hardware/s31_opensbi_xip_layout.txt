ESP32-S31 OpenSBI Flash XIP layout

Goal:
- Keep OpenSBI text/rodata in flash and execute through the flash MMU window.
- Keep OpenSBI writable data, BSS, scratch, stack, and heap in HP SRAM.
- Keep the OpenSBI partition size fixed at 512KB.

Address contract:
- The ESP-IDF second-stage bootloader override maps the opensbi partition for
  instruction access at 0x40030000.
- OpenSBI must be linked with FW_TEXT_START=0x40030000.
- OpenSBI writable sections are linked with FW_RW_START=0x2F040000.
- The second-stage bootloader maps OpenSBI and Linux directly with
  mmu_hal_map_region(); it does not rely on bootloader_mmap() returning the
  final XIP virtual address.

Partition image format:
- Offset 0: raw OpenSBI fw_jump.bin.
- Next byte after fw_jump.bin: OpenSBI FDT blob.
- Padding: zero bytes up to partition_size - 4.
- Last 4 bytes of the 512KB partition: little-endian uint32 file offset of the FDT blob.

Bootloader handoff:
- The factory app is intentionally empty. All Linux handoff work happens inside
  bootloader/bootloader_components/main, which overrides ESP-IDF's default
  second-stage bootloader main component.
- The second-stage bootloader reads the last 4 bytes of the opensbi partition.
- It computes fdt_addr = mapped_opensbi_base + fdt_offset.
- Before mret, the second-stage bootloader initializes PSRAM with the ESP-IDF
  S31 PSRAM device driver path, maps PSRAM through the MMU, and copies the
  initramfs partition to the 0xC0800000 non-cache PSRAM alias.
- Before mret, the second-stage bootloader programs permissive CPU_APM, HP_APM, and
  HP_MEM_APM regions and enables the corresponding function control paths so
  OpenSBI can access SRAM, peripherals, flash, and PSRAM through the existing
  IDF setup.
- Before mret, the second-stage bootloader clears mstatus.MPRV and installs a single
  full-space locked PMP entry in pmpcfg0/pmpaddr0, while clearing pmpcfg1..3
  and pmpaddr1..15.
- Before mret, the second-stage bootloader writes mie=0 and leaves mstatus.MIE/MPIE clear.
  OpenSBI enters M-mode with global interrupts disabled; it must enable and
  route interrupts itself after replacing the bootloader/IDF state.
- The second-stage bootloader does not run pre-handoff test probes or install a temporary
  trap logger; OpenSBI must replace the inherited boot state itself.
- It jumps to mapped_opensbi_base in M-mode with:
  - a0 = 0
  - a1 = fdt_addr

Build-time checks:
- The generated fw_payload.bin must be exactly 512KB.
- OpenSBI + FDT must fit in the first 512KB - 4 bytes.
- Link-time assertions require FW_TEXT_START=0x40030000 and FW_RW_START in HP SRAM.

OpenSBI XIP relocation rule:
- fw_base.S copies .data from flash LMA to FW_RW_START before the boot lottery.
- The R_RISCV_RELATIVE relocation loop then patches pointers inside .data, for
  example platform.ops at platform + 0x60.
- Do not copy .data a second time after relocation. Doing so restores the raw
  flash image, zeros relocated pointers, and can fault on a null load in
  sbi_platform_ops().

Early fault note:
- OpenSBI's first instructions copy .data from flash LMA to FW_RW_START.
- A store/AMO access fault before the OpenSBI banner usually points at handoff
  protection state, not the FDT offset. Check APM FUNC_CTRL, PMP, and MPRV.

12.2
Interrupt Terminology in ESP32-P4
The following terms related to interrupts are defined in the context of the ESP32-P4 Technical Reference
Manual to help readers better understand this document:
12.2.1
Interrupt
An interrupt is an event or condition that temporarily suspends the CPU’s current execution to execute an
interrupt service routine (ISR). This mechanism enables the CPU to respond immediately to important
events.
The ESP32-P4 Technical Reference Manual uses the term “interrupt” in a broader sense to refer to both
interrupt signal and interrupt source.
12.2.2
Interrupt Signal/Interrupt Source
Both interrupt signal and interrupt source describe the same signal from different points in the system.
From the perspective of peripherals, interrupt signals are generated by the peripheral’s internal interrupt
sources and are sent to the interrupt matrix.

Chapter 12 Interrupt Matrix
From the perspective of the interrupt matrix, it receives the interrupt signals sent from peripherals and
considers them interrupt sources. The interrupt matrix then outputs CPU peripheral interrupt signals to the
CPU.
From the perspective of the CPU, the interrupt signals from the interrupt matrix become interrupt sources and
are sent to the CPU core together with the core local interrupt sources.
